- Букон-выгрузка обрывается **19.07.2026**. Дни ≥ 20.07 ведёт сама CRM вживую (админ) — их НЕ трогать импортами.

## 1. Модель денег (СЛОВА БОССА — это закон)
- **Цена визита = фиксированная работа + материал по расходу.** Работа фикс (прайс услуги), материал сколько ушло.
- **Всё что пограмово (краски за грамм: eterna, extremo, SOCOLOR, sync, окисник, фарба, invidia, тонер, порошок, бонд, освітлювач) — это МАТЕРИАЛ**, он сидит в цене визита, отдельной строкой в кассу НЕ пробивается.
- **Штучное (шампунь, маска, кондиционер, ампула) — товар.**
- Биовыравнивання/ботокс/кератин — НЕ окрашивание: тратят свой материал в цене визита, но это не краски eterna.

## 2. Касса за день (итоговая формула)
```
Каса за день = Σ визитов(услуги, done) + Σ розничных продаж товара
```
- Услуги = cash_operations.category='sale_service', ref_type='appointment' — по одной на каждый done-визит, amount = цене Букона.
- Розница = cash_operations.category='sale_product', ref_type IS NULL.
- Материалы к визитам (appointment_materials) списываются со склада, но В КАССУ ОТДЕЛЬНО НЕ идут (краска в цене визита). Материалы-строки в кассе = задвоение, удалять.

## 3. Классификация продажи (sales2.pdf) → РОЗНИЦА или МАТЕРИАЛ
Продажа = **МАТЕРИАЛ** (в цене визита, НЕ в кассу-розницу) если:
  `клиент был в этот день на ОКРАШИВАНИИ` **И** `товар НЕ лечебный-домой`.
Иначе — **РОЗНИЦА** (заводим sale_product ref NULL, идёт в кассу за день).

Признаки:
- **Окрашивание** (regex по услугам визита из cash_report.pdf): `фарбув|тонув|освітл|airtouch|air touch|babyl|мелірув|балаяж|шатуш|розтяжк|деколор|блонд|complex|colorplex|змив волосс`.
  - НЕ включать біовирівнювання/ботокс/кератин (у них свой материал, не краски).
- **Лечебное-домой (всегда розница, даже клиенту на окрашивании):** `проти випадіння|puroxine|medavita|dr.?sorbie|додому`.
- Клиент сматчивается по ПОЛНОМУ имени (norm: lower, только буквы+пробел). НЕ обрезать до первого слова — иначе мастер «Евеліна манікюр» путается с клиентом «Евеліна».

Логика (из /tmp/final-retail.mjs, скопирована в ops/bukon/):
```js
const onColor = clientsOnColoringThatDay.has(norm(client));
const HOME = /проти випадіння|puroxine|medavita|dr.?sorbie|додому/i;
const isMaterial = onColor && !HOME.test(name);   // true → НЕ в кассу
```

## 7. Предел (честно)
- Суммы по каждому дню = Букон до копейки; услуги выверены поштучно (касса=визит).
- Признак «розница/материал» ПО КАЖДОЙ позиции Букон в выгрузку не отдаёт — выведен правилом §3. Суммы дня точны, построчная буконовская сверка каждой продажи невозможна (нет данных). Для 100% построчно нужна выгрузка продаж с колонкой «розница/матеріал» или привязкой к визиту.
