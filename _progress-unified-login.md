# Фича: единый вход мастера (один логин → все его салоны)
Заказ Босса 16.07.2026. Цель: мастер работает в N салонах — входит ОДИН раз, выбирает салон, переключается без разлогина.

## Текущая модель (проверено)
- users: UNIQUE(tenant_id, phone) — аккаунт привязан к салону. Один человек в 2 салонах = 2 записи users (разные tenant_id, тот же телефон).
- routes/auth.js findUserByIdentifier(): ищет по phone/email БЕЗ tenant, возвращает LIMIT 1 (первый). → мастер видит только один салон.
- Изоляция данных (RLS+app_tenant) — не трогаем, остаётся как есть.

## План (Option B — переиспользуем per-tenant аккаунты, минимум риска)
1. findAllUsersByIdentifier() — вернуть ВСЕ совпадения по телефону/email (не LIMIT 1). НОВАЯ функция, старую не трогаем.
2. Новый эндпоинт POST /api/auth/login (или /login-multi): verifyPassword против каждого совпадения → список салонов, где пароль подошёл.
   - 0 → invalid-credentials
   - 1 → как сейчас, сразу сессия
   - >1 → отдать salon-picker (tenant name+role) + короткоживущий pick-token
3. POST /api/auth/pick-tenant {pick_token, tenant_id} → выдать реальную сессию для выбранного салона.
4. Внутри приложения — «Змінити салон» (переключение без пароля в рамках уже подтверждённых салонов сессии).
5. UI: экран-picker после входа если салонов >1; в шапке — переключатель салона.

## РАЗВИЛКА ДЛЯ БОССА (безопасность — его решение)
Как подтверждать доступ к салонам при едином входе:
- A) Показывать ТОЛЬКО салоны, где подошёл введённый пароль (мастер ставит один пароль везде — удобно; разные пароли → увидит только совпавшие). Проще, безопаснее.
- B) Показывать ВСЕ салоны с этим телефоном, доступ по подтверждению (SMS/повторный пароль на каждый). Сложнее, но телефон = единая личность.
Рекомендация: A (один пароль → все салоны, ставит сам мастер). Меньше трения, изоляция цела.
