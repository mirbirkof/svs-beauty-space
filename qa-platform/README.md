# SVS CRM — Autonomous AI QA Platform

Непрерывная система AI-агентов, которая сама ищет баги, подтверждает их, перепроверяет после
исправления и расширяет покрытие CRM. Loop mode 24/7.

## Архитектура
- `lib/orchestrator.js` — распределяет агентов, копит покрытие, не повторяет сценарии впустую,
  объединяет результаты, **регрессия**: каждый цикл перепроверяет открытые баги и закрывает
  ТОЛЬКО с доказательством (повторный сценарий = 0 нарушений).
- `lib/registry.js` — реестр багов (полный формат ТЗ: id, severity, module, role, scenario,
  steps, expected, actual, logs, sql, evidence, cause, fix, status). Дедуп по сигнатуре.
  Статусы: `open` / `reopened` / `closed` (с proof) / `manual` (нельзя автотестить).
- `lib/crm.js` — единая точка доступа к CRM: HTTP (admin-токен) + прямой SQL для верификации.
- `agents/*` — специализированные агенты, каждый возвращает `{scenarios, bugs, coverage}`.

## Агенты (рабочие сейчас, read-only, безопасны для прод-БД)
- `finance-reconciler` — сверка cash-контура, сироты платежей, отрицательная касса → Critical Bug.
- `data-integrity` — сироты записей, отрицательные остатки, битые статусы, формулы без клиента.
- `eventbus-checker` — упавшие/зависшие domain_events.
- `api-contract` — схемный контракт таблиц под эндпоинты (битый контракт = 500).
- `security-probe` — tenant-изоляция, wildcard-роли, сила admin-токена.

## Безопасность (КРИТИЧНО)
Деструктивные агенты (load: десятки тысяч записей; security: SQLi/XSS/CSRF/IDOR/race/двойные
платежи через HTTP; write-heavy ролевые сценарии) **НЕ запускаются против боевой БД салона** —
это уничтожило бы реальные данные. Они включаются ТОЛЬКО при изолированном таргете:
- `QA_TENANT_ID` — отдельный тенант для тестовых данных, ИЛИ
- `QA_STAGING_API` — staging-сервер.
Без них режим = `safe` (только read-only сверки). Активные атаки честно помечаются `needs-manual`
с причиной (по ТЗ: «если нельзя проверить автоматически — пометить с объяснением»).

## Запуск
```
node run.js            # один цикл (для крона)
node run.js --loop     # непрерывно 24/7 (пауза = QA_COOLDOWN_MS, по умолч. без паузы)
node run.js --report   # сводка по багам и покрытию
```

## Loop mode
Каждый цикл: меняется порядок агентов (детерминированный shuffle по номеру цикла),
сценарии не повторяются чаще `scenarioDedupWindowH` (12ч), номер цикла растёт, состояние
в `data/cycle-state.json`. Никогда не выполняет идентичный цикл дважды.

## Расширение покрытия
- Новые правила целостности → массив `RULES` в `data-integrity.js`.
- Новые финансовые сверки → `finance-reconciler.js`.
- Новые ролевые/workflow-агенты → файл в `agents/` + имя в `orchestrator.loadAgents()`.

## Что требует изолированного таргета (roadmap, сейчас needs-manual)
Load-тесты, активные security-атаки, полные ролевые workflow с мутациями (Лид→...→Возврат),
UI-тесты (Playwright), HTTP-контракт всех эндпоинтов. Все они ждут QA-тенанта/staging.
