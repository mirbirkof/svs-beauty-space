## [ДОБАВЛЕНО 16.07] ПОДТВЕРЖДЕНИЕ ТЕЛЕФОНА ПРИ РЕГИСТРАЦИИ (Босс: обязательно, бесплатно)
ПРОБЛЕМА: signup создаёт тенант без проверки телефона → фейки/спам.
РЕШЕНИЕ (бесплатно, механика уже есть): Telegram request_contact.
Наш стек уже умеет: booking.js:288 просит клиента "Поділитись номером" (request_contact:true) → TG отдаёт верифицированный phone_number.
ФЛОУ регистрации с верификацией (аддитивно, не ломая текущий signup):
1. Форма (салон, телефон, пароль) → НЕ создаём тенант сразу. Пишем pending_signup(token, данные, phone, expires 15мин).
2. Экран "Підтвердіть номер" + кнопка/дип-линк t.me/<SalonBot>?start=sv_<token>.
3. Бот при /start sv_<token> шлёт кнопку "📱 Підтвердити номер" (request_contact). Юзер делится контактом.
4. Бот сверяет contact.phone_number с телефоном из pending → совпал → помечает token verified, шлёт "Готово, поверніться в браузер".
5. Веб поллит GET /api/public/signup-status?token= → verified → вызывает createTenant → вход.
ЗАЩИТА: телефон из TG contact ДОЛЖЕН совпасть с введённым (иначе подмена). Rate-limit на pending. TTL 15мин. Один token — один тенант.
АЛЬТЕРНАТИВА (если нет своего бота на этапе регистрации): общий платформенный бот @SvsSignupBot для верификации, дальше салон подключает свой.
СТАТУС: ГОТОВО и в проде (rev 339dc56), строили аддитивно + тест 3037. Это BLOCKER перед публичным открытием регистрации (иначе спам-тенанты).
