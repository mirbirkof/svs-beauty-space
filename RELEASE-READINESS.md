# CRM SVS — Единый критерий готовности (source of truth)

Обновляется только здесь. Любой вопрос «сколько %» — ответ отсюда, не с потолка.

## Почему раньше прыгало (80 / 95 / 98)
Мерили РАЗНОЕ. Есть ДВА продукта — две цифры:
- **A. Для салона Босса (один тенант)** — работает сейчас, каждый день.
- **B. Для ПРОДАЖИ чужим салонам (multi-tenant SaaS)** — строже: нужны юр-слой,
  биллинг подписки, надёжность под чужой нагрузкой.
Цель Босса = B (продавать). Дальше считаем ТОЛЬКО по B. A даём справочно.

## Конечное представление = 7 столбов. Готово продавать = все 7 зелёные.

| # | Столб | Что значит «зелёный» | Статус B |
|---|-------|----------------------|----------|
| 1 | Изоляция салонов | Данные одного салона НИКОГДА не видны другому | 🟢 ГОТОВО |
| 2 | Деньги верны | ЗП/склад/касса/бонусы сходятся гривна-в-гривну | 🟢 ГОТОВО (осн.) |
| 3 | Не падает / фейловер | Авария сервера/базы → сам переключается, данные целы | 🟢 ГОТОВО |
| 4 | Онбординг салона | Новый салон сам регится и доходит до 100% без нас | 🟡 ПОЧТИ |
| 5 | Юридика / GDPR | Согласие ПД, шифрование, право на удаление, оффбординг | 🟡 ПРОВЕРЯЕТСЯ |
| 6 | Биллинг подписки | Салон платит ТЕБЕ за SaaS: тариф, триал, блок за неоплату | 🔴 ПРОБЕЛ (проверяется) |
| 7 | Надёжность интеграций | Retry для Telegram/Mono, подпись вебхуков, health по БД | 🟡 ПОЧТИ |

Зелёных сейчас: 3 из 7 твёрдо + 3 «почти» + 1 пробел.

## Единая цифра (по столбам, а не на глаз)
- **A (свой салон): ~92%** — работает, критичное закрыто, мелкие рассинхроны.
- **B (продажа чужим): ~85%** — было 78-80% (07.07), выросло после миграций
  согласия/шифрования ПД (230/231). Осталось не «настройки», а реальная работа.

## План до 100% по B (что чинить, по приоритету)
### Блок 1 — деньги-мелочи (дни)
- Mono-комиссия эквайринга в расходы P&L; аванс ЗП — исключить двойной учёт.
- recordCashIn при prepaid — не best-effort (визит оплачен, а в кассе пусто).
- booking start==end запретить; qty заказа — верхний лимит; TZ Киев/UTC у границ суток.
### Блок 2 — надёжность (1-2 нед)
- Retry/backoff для Telegram/Mono при 429/timeout (не терять уведомления о платежах).
- Подпись Viber/Mono вебхуков (защита от подделки).
- Health-check пингует реальную БД; кроны не глотают ошибки (алерт при падении).
- Пул БД под дашбордом (10-15 виджетов) — не исчерпывать.
### Блок 3 — юридика/GDPR (проверить закрытие, добить 1-2 нед)
- Согласие ПД реально собирается в UI при регистрации клиента.
- Телефоны/аллергии реально шифруются (не пустые колонки).
- Право на удаление клиента (hard-delete), оффбординг салона, политика/DPA при signup.
### Блок 4 — биллинг подписки SaaS (1-2 нед) — БЕЗ него продавать нельзя
- Тарифы, оплата подписки салоном, триал, автоблок за неоплату.
- (feature-gate по фичам уже есть — нет самого приёма денег за подписку.)

## Определение «ГОТОВО ПРОДАВАТЬ»
Все 7 столбов 🟢 + прогон E2E чужого салона: регистрация → оплата подписки →
работа → уход, без нашего вмешательства. Оценка до этой точки: **2-4 недели**.
