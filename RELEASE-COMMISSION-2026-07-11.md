## 2. МОЖНО ЛИ ПРОДАВАТЬ

### **NO-GO для внешних клиентов.**

**Пилот 1–3 дружественных салона** — допустим **только после** закрытия блокеров финансовой целостности и бронирования (Priority 1–3 ниже), с явным предупреждением партнёров об отсутствии GDPR-комплаенса.

**Причины NO-GO:**
- Двойное бронирование реально при любой нагрузке > 1 rps (EXCLUDE-констрейнт уничтожен migration 239)
- 2FA bypass: владелец включил защиту, думает что защищён — она не работает
- Финансовая целостность: Mono-платёж удаляется hard DELETE без сторно, ЗП мастера с fixed_per_day занижается автоматически каждое 1-е и 16-е число
- GDPR: 4 blocker + tenant isolation не аудирован = продавать EU/UA salons с персональными данными клиентов незаконно
- Tenant isolation: пласт не покрыт — неизвестно что сломано в multi-tenant изоляции

## 5. СЛЕПЫЕ ЗОНЫ — ЧТО КОМИССИЯ НЕ ПОКРЫЛА

| Зона | Риск |
|---|---|
| **Tenant isolation (pct=null)** | Неизвестно, может ли тенант A читать данные тенанта B через прямые запросы. Без этого аудита SaaS продавать нельзя в принципе |
| **API QA (pct=null)** | Нет покрытия regression-тестами — каждый фикс может сломать другое |
| **Нагрузочное тестирование** | Race condition блокеры (B1, B2, E1) воспроизводятся при > 1 rps. Пороги неизвестны |
| **Backup restore** | S3-ключи не настроены → реальные бэкапы не создаются. Процедура restore ни разу не проверялась |
| **Monobank webhook security** | Проверяется только X-Sign из env. Не аудировалось: идемпотентность при дублях, обработка refund-событий, таймаут верификации |
| **Admin panel: 100+ маршрутов не покрыты api-qa** | Пласт «api-qa» = null. Нет данных об авторизации на отдельных admin-эндпоинтах |
| **Penetration testing** | Аудит статический (grep + code review). Не проводилось: реальный XSS exploit, SQL injection через параметры, IDOR через смену tenant в заголовке |
| **Mobile-клиент (если есть)** | Не упомянут в аудите |
| **Миграционный путь данных** | При G4 (phone_enc → миграция) нет плана отката для prod-данных уже зашифрованных записей |

**Резюме председателя:** Продукт технически живой для одного доверенного салона (своего), но на релиз в таком состоянии не выходит. 15 блокеров — это не "подчистить перед релизом", это системные дыры в финансах, безопасности и праве. GDPR-пласт на 28% с 4 блокерами означает, что работа с персональными данными клиентов юридически не оформлена. Tenant isolation = terra incognita. **Рекомендация: 3 недели спринт по группам A–G, повторный аудит пластов tenant-isolation и api-qa, после — ограниченный пилот.**
