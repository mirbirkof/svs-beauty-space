## КРИТИЧЕСКИЕ (7)
1. Двойная оплата визита. schedule.js:1256-1300 /pay — идемпотентность через SELECT-then-INSERT
   без блокировки и без unique(ref_type,ref_id). Два параллельных /pay → двойной приход в кассу.
2. Перерасход сертификата. gift-certificates.js:387-409 /use — read→check→UPDATE без FOR UPDATE.
   Два одновременных /use спишут больше номинала (lost update).
3. Бесконечное начисление бонусов. lib/bonus.js:74-122 accrue без идемпотентности по source_id —
   один оплаченный визит можно начислить бонусами сколько угодно раз.
4. Inbox (омниканальность) мёртв целиком. omnichannel.js:108 GET /api/omni/conversations →
   500 "column tenant_id is ambiguous" (после JOIN clients). Список диалогов не грузится,
   вся страница inbox.html нерабочая. Подтверждено на проде.
5. KPI-дашборд сотрудников мёртв. kpi.js использует appointments.duration_min — колонки НЕТ в
   prod-БД. /api/kpi/employees и /leaderboard → 500. Вкладка KPI + лидерборд пустые.
6. Утечка закупочных цен. material-norms.js — ВСЕ GET без requirePerm. Аноним получает wholesale,
   себестоимость, маржу услуг через /api/material-norms/reports/profitability.
7. #108 (заметка владельца) пикер услуг. index.html:8279 — нативный <select> без size и width:100%.
   Свёрнутый показывает только верхнюю строку ("только верхний"), длинные названия рвут окно
   ("не влазит"). Плюс на Save берётся select.value = верхняя опция → молча не та услуга в записи.

- #108 переполнение окна по ширине (длинные названия услуг растягивают select за max-width:480px).
- #108 молчаливое сохранение не той услуги при активном фильтре поиска.

## ПОЧЕМУ ШТАТНЫЕ ТЕСТЕРЫ ЭТОГО НЕ НАШЛИ
QA-платформа проверяет ЦЕЛОСТНОСТЬ уже записанных данных (склад не отрицательный, касса сходится
постфактум) и статику. Она не запускает КОНКУРЕНТНЫЕ сценарии (два /pay одновременно) и не читает
runtime-500 на роутах, которые не в её списке. Дыры идемпотентности видны только под гонкой,
а мёртвые разделы — только при реальном GET с токеном. Нужен слой e2e-тестов с параллелизмом.
